## 3. 项目案例页规划

### 3.1 项目一：矿山教育系统

页面定位：

> 企业培训场景下的微服务学习闭环系统。

建议路由：

`/projects/mine-education-system`

页面结构：

- 项目概览：面向企业安全培训，提供课程学习、视频进度跟踪、章节考试、学习记录统计。
- 我的职责：学习服务表结构、接口开发、学习状态流转、Redis 缓存方案。
- 架构图：Gateway、用户服务、课程服务、学习服务、MySQL、Redis、RabbitMQ。
- 核心业务流：课程开始学习 -> 小节播放 -> 进度上报 -> Redis 缓存 -> 完播/延迟同步 -> 更新课表状态。
- 难点复盘：高频进度上报导致数据库压力；视频与考试两类小节状态不同；异常和日志治理。
- 可演示模块：视频学习进度模拟器、学习状态机、缓存同步链路。
- 面试讲解重点：为什么用 Redis 缓存、什么时候同步 MySQL、如何保证学习状态正确、异常日志如何帮助排查问题。

推荐可交互 Demo：

- 视频进度条模拟：每秒上报进度。
- Redis 面板：显示当前缓存中的学习进度。
- MySQL 面板：显示最终落库状态。
- 延迟任务按钮：模拟定时同步。
- 状态机展示：未开始、学习中、已完成、考试通过。

## 7. 面试表达包装

### 1 分钟自我介绍版本

> 我是山西大学软件工程专业本科生，主攻 Java 后端开发。实习中参与过公司产品中心系统的后端迭代、接口联调、问题排查、慢 SQL 优化、邮件告警和特殊格式文件解析，也加入过 AI 项目小组，接触过私有化 AI 平台、RAG、SQL 智能问答等工程化接入。项目上，我重点做过企业培训场景下的矿山教育系统，围绕学习记录、视频进度上报、章节考试和学习状态流转完成后端开发，并针对高频进度上报设计了 Redis + 延迟任务缓存方案，减少了重复写入并优化接口响应。

### 项目讲解主线

矿山教育系统：

> 这个项目最值得讲的是学习进度高频上报。问题是视频播放过程中如果每次进度都直接写 MySQL，会造成大量重复写入和接口响应压力。我设计的方案是首次播放创建学习记录，播放过程中优先写 Redis，完播或延迟任务触发后再同步到 MySQL，同时根据视频/考试小节类型驱动课程状态更新。

私有化 AI 平台：

> 这个项目更偏 AI 应用工程化，重点不是训练模型，而是把 MaxKB、SqlBot、RAG、平台配置和部署链路接入企业内部环境，解决本地化部署、数据隔离、知识库检索和自然语言查询数据库的问题。
