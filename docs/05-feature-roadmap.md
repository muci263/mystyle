# 功能路线图

本文件记录后续可以逐步实现和微调的增强想法。当前阶段先记录与分级，不要求一次性全部开发。

## 1. 总体方向

项目最终目标：

> 一个面向秋招的智能个人作品集系统，包含可扩展经历资产库、JD 智能适配、定制分享链接、工作模块复现实验室、项目证据链和面试模式。

核心原则：

- 功能服务于秋招面试，不做无意义炫技。
- 经历、项目、技能、模块 Demo 都要数据化和可扩展。
- LLM 只能基于真实经历做表达重组，不编造经历。
- 每个重点项目最好都有证据链、可演示模块和面试讲解点。
- 第一版先做闭环，后续逐步增强视觉、交互和自动化。

## 2. P0：第一期核心功能

### 2.1 项目证据链

功能：

- 将项目中的业务问题、我的方案、技术实现、结果指标、对应 Demo、面试追问串起来。

价值：

- 提升可信度。
- 让项目经历不只是描述，而是有证据、有指标、有演示。

示例：

```text
业务问题：视频进度高频上报导致数据库写入压力大
解决方案：Redis + 延迟任务
结果指标：重复写入减少约 70%，接口响应从 1.5s 降到 500ms
对应 Demo：视频播放学习链路复现
可追问点：缓存一致性、幂等、状态流转
```

### 2.2 定制分享链接

功能：

- 为不同公司、岗位或 JD 生成不同展示链接。
- 打开后首页、项目排序、模块推荐、个人介绍重点不同。

示例：

```text
/share/java-backend
/share/ai-platform
/share/fullstack
```

价值：

- 投递不同岗位时展示不同重点。
- 与 JD 智能适配器天然联动。

### 2.3 面试模式

功能：

- 根据目标岗位或 JD 生成面试准备页。

内容：

- 30 秒自我介绍
- 1 分钟自我介绍
- 3 分钟项目讲解
- 推荐项目顺序
- 推荐打开的 Demo
- 面试官可能追问
- 回答要点

价值：

- 面试前快速准备。
- 面试中可以作为导览页。

### 2.4 JD 智能适配器

功能：

- 输入岗位 JD。
- 分析岗位关键词。
- 匹配个人技能、实习、项目和模块 Demo。
- 生成岗位适配版个人介绍、项目排序、项目强调重点和风险提示。

价值：

- 让作品集不再是固定展示，而是能根据岗位动态调整。

### 2.5 工作模块复现实验室

功能：

- 将真实工作中的关键模块抽象成可交互 Demo。

第一期重点：

- 视频播放学习链路复现。
- Redis 学习记录缓存同步。
- Agent 工作流复现。

价值：

- 面试官能直接看到你参与过的工作如何运转。

## 3. P1：第二期增强功能

### 3.1 技术能力地图

功能：

- 将技能与项目、模块、指标、面试讲解点关联。

示例：

```text
Redis
  关联项目：矿山教育系统
  关联模块：视频进度缓存同步
  指标：重复写入减少约 70%
```

价值：

- 技能不再是孤立标签，而是有项目支撑。

### 3.2 项目架构交互图

功能：

- 在项目页中展示可点击架构图。
- 点击节点查看职责、技术点、数据流、设计原因。

示例节点：

- Gateway
- Learning Service
- Redis
- MySQL
- RabbitMQ
- LLM
- RAG Retriever

价值：

- 增强项目页的工程感和讲解效率。

### 3.3 代码片段与接口调试台

功能：

- 展示关键接口、请求参数、响应示例、后端处理步骤。
- 对实验室接口提供在线 Try It。

价值：

- 强化后端接口设计能力展示。

注意：

- 只开放实验室接口。
- 后台管理和敏感接口必须鉴权。

### 3.4 项目复盘文章系统

功能：

- 每个重点项目或模块配复盘文章。

文章结构：

- 背景
- 问题
- 方案
- 实现
- 踩坑
- 优化
- 面试追问

价值：

- 展示总结能力。
- 方便面试前复习。

### 3.5 本地管理员后台访问控制（待讨论）

功能：

- 增加后台入口，但不做登录注册按钮。
- 后台入口只在管理员可访问环境展示。
- 后端同时保护 `/admin/**` 页面和 `/api/admin/**` 接口，避免只做前端隐藏。

候选方案：

- 本地 IP 白名单：默认只允许 `127.0.0.1`、`::1` 或可信内网访问。
- 可选访问 token：后续部署到公网时，通过环境变量配置一次性密钥或 Cookie。
- 非管理员访问时隐藏后台入口，并对后台页面/API 返回 404 或 403。

待确认：

- 是否只服务本地开发环境，还是未来部署后也要远程访问。
- 是否需要固定公网 IP 白名单。
- 后台被拒绝访问时使用 404 隐藏存在感，还是 403 明确提示无权限。

## 4. P2：第三期扩展功能

### 4.1 访问分析

功能：

- 页面 PV。
- 项目点击。
- Demo 点击。
- 分享链接访问。
- 页面停留时间。

价值：

- 知道投递链接是否被查看。
- 展示埋点、日志和统计分析能力。

注意：

- 第一版只做匿名统计。
- 避免过度追踪访问者。

### 4.2 简历版本管理与 PDF 导出

功能：

- 维护多个简历版本。
- 支持页面预览。
- 后续支持 PDF 导出。

版本示例：

- Java 后端版
- AI 应用版
- 全栈版
- 实习投递版
- 校招正式版

价值：

- 方便不同岗位投递。
- 与 JD 适配器、定制分享链接联动。

### 4.3 更复杂的 Agent 编排

功能：

- 从模拟节点升级为真实工具调用。
- 接入 Spring AI / LangChain4j。
- 支持节点拖拽配置。
- 支持流程保存和运行历史。

价值：

- 进一步展示 AI 应用工程化深度。

## 5. 实施建议

第一阶段先完成：

1. 项目基础架构。
2. 可扩展内容模型。
3. 首页、项目页、模块实验室骨架。
4. 视频播放学习链路 Demo。
5. JD 智能适配器基础版。
6. 定制分享链接基础版。

第二阶段增强：

1. 项目证据链完整展示。
2. 面试模式。
3. Agent 工作流复现。
4. 技术能力地图。

第三阶段完善：

1. 访问分析。
2. 简历版本管理。
3. PDF 导出。
4. 更复杂的部署和监控展示。
