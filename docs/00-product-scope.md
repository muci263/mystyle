# 高级个人作品展示集 - 产品与工程规划

## 1. 项目定位

这个项目不是静态简历页，而是一个真实上线运行的全栈个人展示系统：

- 面向秋招面试官，第一眼展示审美、工程完整度和技术成熟度。
- 面向技术面试，能够现场演示核心模块、讲清楚架构设计与实现细节。
- 面向长期维护，支持后续新增实习、项目、文章、模块 Demo 和访问数据。
- 面向不同岗位 JD，能够基于已有经历动态生成更匹配的个人介绍、项目讲解重点和页面展示内容。

一句话定位：

> 高级个人作品集 + 在线简历 + 项目复盘系统 + 业务模块演示实验室 + JD 智能适配器 + DevOps 上线样板。

## 2. 设计基调

视觉关键词：

- 高级、克制、清晰、有工程感。
- 避免常见 AI 紫蓝渐变、玻璃拟态泛滥、装饰性光斑。
- 使用中性色、精致排版、真实内容密度、细腻动效。
- 首屏直接展示个人身份和代表作品，不做空洞营销页。

建议方向：

- 主色：近黑、象牙白、冷灰、钴蓝或石墨绿中的一种作为强调色。
- 字体：中文优先清晰可读，英文标题可配合几何无衬线。
- 动效：页面转场、滚动揭示、卡片 hover、命令面板式跳转，少量但精准。
- 内容组织：像成熟 SaaS 产品的控制台 + 高端作品集，而不是普通博客模板。

## 3. 推荐技术栈

### 前端

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- shadcn/ui 或 Radix UI
- Motion / Framer Motion
- Lucide Icons
- ECharts / Recharts
- MDX，用于项目复盘、技术文章和经历详情

### 后端

两个候选方向：

- Java 方向：Spring Boot + Spring Security + MyBatis Plus / JPA
- Node 方向：NestJS + Prisma / TypeORM

如果秋招主投 Java 后端，建议选 Spring Boot；如果主投前端或全栈，建议选 NestJS。后端不只是内容接口，要展示工程深度。

### 数据与基础设施

- MySQL / PostgreSQL
- Redis
- JWT + RBAC
- Swagger / OpenAPI
- Docker / Docker Compose
- Jenkins 自动化部署
- Nginx 反向代理
- 云服务器真实上线

## 4. 核心页面

### 4.1 首页

目的：10 秒内让面试官知道你是谁、强在哪里、有什么可点开看的。

模块：

- Hero：姓名、求职方向、核心标签、简历入口、GitHub、邮箱。
- Signature Work：精选 2-3 个代表项目。
- Experience Snapshot：实习经历时间轴。
- Capability Matrix：前端、后端、工程化、DevOps 能力矩阵。
- Live Modules：可交互业务模块入口。
- Recent Notes：技术文章或项目复盘入口。

### 4.2 在线简历

目的：替代 PDF 简历的结构化版本，同时保留 PDF 下载。

模块：

- 教育背景
- 实习经历
- 项目经历
- 技术栈
- 荣誉 / 证书 / 开源
- PDF 下载
- 一键复制邮箱 / 微信占位 / 联系方式

### 4.3 项目详情页

目的：把每个项目包装成可讲的工程案例。

标准结构：

- 项目背景
- 业务目标
- 技术栈
- 我的职责
- 系统架构图
- 核心模块
- 难点与方案
- 性能 / 稳定性 / 安全性考虑
- 可交互 Demo
- 复盘与改进

### 4.4 模块实验室

目的：重现你参与过的重要业务模块，让“我做过”变成“我能现场演示”。

首批建议模块：

- RBAC 权限管理：用户、角色、菜单、按钮权限。
- 数据看板：筛选、图表、指标卡、趋势分析。
- 文件上传：分片上传、进度条、秒传/断点续传说明。
- 表单流程：多步骤表单、校验、草稿、状态流转。
- 搜索过滤：多条件筛选、排序、分页、URL 状态同步。
- 消息通知：站内消息、未读数、操作反馈。

结合当前简历，模块实验室要优先复现真实工作中的核心链路：

- 视频播放学习链路：播放器事件、进度上报、Redis 缓存、完播同步、状态更新。
- Redis 学习记录缓存同步：高频写入削峰、延迟任务、MySQL 落库。
- Agent 搭建与流程编排：输入、意图识别、工具路由、RAG/SQL 查询、模型生成、结果校验。
- SQL 智能问答：自然语言到 SQL，再到结果解释。
- RAG 知识库检索：切块、向量写入、召回、回答生成。

模块实验室需要支持可扩展模式：

- 每个复现模块都有结构化 ModuleDemo 元数据。
- 页面使用统一模块外壳渲染业务背景、技术点、链路、日志和面试讲解点。
- 具体交互由 DemoRenderer 按 `demoType` 分发。
- 后续新增工作经历时，可以新增模块配置并绑定 Project、Experience、Skill。
- JD 智能适配器可以根据模块关键词自动推荐最适合展示的模块。

### 4.5 技术架构页

目的：专门给技术面试官看工程能力。

模块：

- 前后端架构图
- 数据库 ER 图
- 接口设计说明
- 鉴权与权限流程
- 缓存设计
- 日志与异常处理
- Jenkins 流水线
- Docker/Nginx 部署结构
- 线上访问链路

### 4.6 后台管理

目的：证明这是可维护的真实系统，而不是硬编码页面。

模块：

- 管理员登录
- 项目管理
- 经历管理
- 技术文章管理
- 模块 Demo 配置
- 简历信息管理
- 访问统计

### 4.7 JD 智能适配页

目的：输入目标岗位 JD 后，由 LLM 基于个人经历生成更适配该岗位的展示内容，帮助投递和面试准备。

模块：

- JD 输入框：支持粘贴岗位描述。
- 岗位解析：提取岗位方向、技术关键词、业务要求、加分项。
- 匹配分析：将 JD 要求与个人技能、实习、项目经历进行匹配。
- 内容生成：
  - 岗位适配版个人介绍。
  - 项目经历排序建议。
  - 每个项目的强调重点。
  - 面试自我介绍 1 分钟 / 3 分钟版本。
  - 可放到首页 Hero 或项目页的动态摘要。
- 风险提示：标记简历中没有支撑的要求，避免生成过度包装内容。
- 保存方案：将某个 JD 的适配结果保存为一个 Profile Variant，后续可再次查看或用于页面预览。

## 5. 后端技术深度设计

第一期后端需要包含：

- 用户认证：登录、JWT、刷新机制。
- 权限系统：RBAC，区分页面权限和按钮权限。
- 内容管理：项目、经历、文章、技能标签 CRUD。
- 文件服务：简历 PDF、项目图片、附件上传。
- 访问统计：PV/UV、项目点击、模块演示访问。
- 接口文档：Swagger/OpenAPI。
- 全局异常处理、统一响应体、参数校验。
- 日志：请求日志、错误日志、关键操作日志。
- JD 适配：岗位 JD 解析、经历匹配、LLM 调用、生成结果保存。

进阶增强：

- Redis 缓存热门项目和首页数据。
- 异步任务记录访问日志。
- 限流、防刷、基础安全头。
- 简单压测报告和性能优化记录。
- LLM 调用缓存、Prompt 版本管理、生成结果审核与回滚。

## 6. Jenkins 自动化部署

目标：GitHub 推送后自动构建、测试、打包、部署到服务器。

推荐流程：

1. GitHub push 到 main。
2. Jenkins webhook 触发流水线。
3. 拉取前端、后端代码。
4. 安装依赖并执行 lint/test/build。
5. 构建 Docker 镜像。
6. 推送镜像或在服务器本地构建。
7. docker compose 更新服务。
8. Nginx 指向新服务。
9. 健康检查通过后完成部署。

需要产出：

- Jenkinsfile
- Dockerfile.frontend
- Dockerfile.backend
- docker-compose.yml
- nginx.conf
- .env.example
- 部署文档

## 7. 第一阶段 MVP

第一期目标是做出可以上线、可以面试演示的最小完整闭环。

包含：

- 首页
- 在线简历页
- 项目列表页
- 1 个完整项目详情页
- 模块实验室中的 RBAC Demo 和数据看板 Demo
- 管理后台登录
- 项目/经历内容管理
- 后端接口与 Swagger
- Docker Compose 本地部署
- Jenkins 自动部署脚本雏形

暂缓：

- 复杂 3D 场景
- AI 聊天助手
- 过多博客系统功能
- 复杂评论/社交功能

## 8. 需要你补充的信息

为了把内容做得像“你的项目”而不是模板，需要补充：

- 目标岗位：前端 / Java 后端 / 全栈 / 其他。
- 技术栈熟练度：React、Vue、Java、Spring Boot、Node、数据库、Redis、Docker、Jenkins。
- 实习经历：公司、时间、岗位、业务、负责内容、成果。
- 项目经历：项目名、背景、技术栈、你负责的模块、难点、结果。
- 最想重现的 2-4 个业务模块。
- 是否已有服务器、域名、GitHub 仓库。
- 是否希望后台管理也做得很完整，还是以展示和演示为主。
